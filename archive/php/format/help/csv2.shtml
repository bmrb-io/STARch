Delimited file format, one residue per line

Generic input formats where each line contains data for one residue and values are separated by either whitespace (spaces and/or tabs: tab-delimited) or commas (CSV: comma-separated values).

First two columns in uploaded file must contain residue label and residue sequence number. Subsequent columns contain values.

First line of the uploaded file must contain column headers: atom names (except for the first two columns whose names are ignored).

Because these are "free-form" formats, the software will not attempt to guess NMR-STAR tags, you will have to select NMR-STAR tag for each column from drop-down lists later on.

Tab-delimited format note: because the software treats consecutive whitespace characters as one you will have to put a non-whitespace marker in place of missing values (e.g. "?" or "n/a"). Otherwise subsequent column(s) will be shifted to the left.

Comma-separated files do not have this problem because they use an explicit field delimiter: comma.
