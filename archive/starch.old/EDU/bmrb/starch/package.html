Design overview

Starch is a Vector of converter classes. Its main method is read( file ) -- it will run an appropriate converter on the file.

Converter is a parent class for all converters, each converter class knows how to parse a particular input format. Available converters are Format1, Format2, Format3 (aka NucleicAcid), Star2, Pipp, and Xeasy.

ErrorList is a Vector of error and warning messages (formatted strings). Each converter adds messages to the list; messages can be printed out, retrieved individually or as one multi-line string.

Data is a Vector of residues. Converters add residues and add atoms to residues, as they parse the input. Residues can be printed out or returned as ulti-line string in one of the available output formats.

Residue is data structure that holds one residue. It contains residue label, type, sequence code (number), and a Vector of atoms.

Atom is data structure that holds one atom.

Nomenmap is a nomenclature mapper class. Its methods operate on residues; the mapper can convert residue labels, expand pseudo-atoms, supply atom types, etc.

Utils class contains a few generally useful static methods.

Messages class contains error and warning messages (as public string constants).

Finally, Main is the command-line interface to Starch. Main is a good example of how to use Starch package in your code.
